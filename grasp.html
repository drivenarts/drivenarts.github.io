Overview

GRASP SDK is a powerful Unreal Engine plugin that enables developers to program and control humanoid robots using familiar Unreal Engine tools and workflows. Built on the foundation of VR training experiences, GRASP brings the power of game development to robotics.

Environment First

Define your space and equipment properties, letting the robot discover and adapt to its environment naturally.

High-Level Tasks

Focus on what needs to be done using familiar Unreal tools, while the SDK handles the complexities of robot execution.

Familiar Tools

Use Blueprints and C++ with the same workflows you know, making robot programming as intuitive as game development.

Robot Integration

Seamlessly integrate with leading robot platforms while focusing on business logic rather than low-level control.

Getting Started

Installation

1. Download GRASP SDK from the Unreal Marketplace
2. Enable the plugin in your project settings
3. Add the following to your project's Build.cs:
   PublicDependencyModuleNames.AddRange(new string[] { "GRASP" });

C++ API

High-Level Task Orchestration

#include "GRASP.h"

// Define the restaurant environment and equipment
class ARestaurantEnvironment : public AActor
{
    GENERATED_BODY()

public:
    // Initialize the environment with known equipment properties
    virtual void BeginPlay() override
    {
        // Register equipment properties that the robot needs to know about
        RegisterEquipment("POS_System", {
            Dimensions: FVector(60.0f, 30.0f, 90.0f),  // Width, Depth, Height in cm
            InteractionPoint: FVector(30.0f, 0.0f, 45.0f),  // Relative to equipment center
            RequiredClearance: 100.0f,  // Minimum distance needed for operation
            InteractionHeight: 90.0f,   // Height of the touch screen
            MaxReachDistance: 80.0f     // Maximum distance for interaction
        });
        
        RegisterEquipment("Coffee_Maker", {
            Dimensions: FVector(40.0f, 50.0f, 75.0f),
            InteractionPoint: FVector(20.0f, 25.0f, 60.0f),
            RequiredClearance: 80.0f,
            CupPlacementHeight: 65.0f,  // Height where cups should be placed
            MaxReachDistance: 70.0f
        });
        
        RegisterEquipment("Donut_Case", {
            Dimensions: FVector(120.0f, 40.0f, 100.0f),
            InteractionPoint: FVector(60.0f, 20.0f, 50.0f),
            RequiredClearance: 90.0f,
            ShelfHeights: {40.0f, 60.0f, 80.0f},  // Heights of each shelf
            MaxReachDistance: 85.0f
        });
    }

private:
    // Map of equipment properties (excluding location)
    TMap EquipmentProperties;
};

// Main robot controller class
class AServiceRobot : public AActor
{
    GENERATED_BODY()

public:
    // High-level service loop
    void ServiceLoop()
    {
        while (IsOperating)
        {
            // Check for customers at counter
            if (IsCustomerWaiting())
            {
                // Handle the customer interaction
                HandleCustomerInteraction();
            }
            
            // Check for pending orders
            if (HasPendingOrders())
            {
                // Process the next order
                ProcessNextOrder();
            }
        }
    }

private:
    // Customer interaction flow
    void HandleCustomerInteraction()
    {
        // Navigate to POS system
        NavigateToEquipment("POS_System");
        
        // Greet customer and get order
        FOrder NewOrder = GetCustomerOrder();
        
        // Process payment
        if (ProcessPayment(NewOrder))
        {
            // Add order to queue
            AddOrderToQueue(NewOrder);
        }
    }

    // Order fulfillment flow
    void ProcessNextOrder()
    {
        FOrder CurrentOrder = GetNextOrder();
        
        // Navigate to required equipment
        for (const FString& Equipment : CurrentOrder.RequiredEquipment)
        {
            NavigateToEquipment(Equipment);
            
            // Perform the required action
            if (Equipment == "Coffee_Maker")
            {
                PrepareBeverage(CurrentOrder.BeverageType);
            }
            else if (Equipment == "Donut_Case")
            {
                RetrieveDonuts(CurrentOrder.DonutTypes);
            }
        }
        
        // Deliver order to customer
        DeliverOrder(CurrentOrder);
    }

    // Navigation helper
    void NavigateToEquipment(const FString& EquipmentName)
    {
        // The actual navigation is handled by the SDK
        // This is just a high-level interface
        FVector TargetLocation = GetEquipmentLocation(EquipmentName);
        NavigateToLocation(TargetLocation);
    }
};

// Example usage in game mode
class ARestaurantGameMode : public AGameModeBase
{
    GENERATED_BODY()

public:
    virtual void BeginPlay() override
    {
        // Spawn the environment
        Environment = GetWorld()->SpawnActor();
        
        // Spawn and initialize the robot
        Robot = GetWorld()->SpawnActor();
        Robot->Initialize(Environment);
        
        // Start the service loop
        Robot->ServiceLoop();
    }

private:
    UPROPERTY()
    ARestaurantEnvironment* Environment;
    
    UPROPERTY()
    AServiceRobot* Robot;
};

Key Concepts

  • Environment Registration: The robot maintains awareness of its environment by registering equipment properties that can be discovered at runtime.
  • High-Level Task Flow: Developers focus on business logic (customer service, order processing) rather than low-level robot control.
  • Modular Design: Complex behaviors are broken down into manageable, reusable functions.
  • Event-Driven Architecture: The service loop continuously checks for new tasks and customer interactions.
